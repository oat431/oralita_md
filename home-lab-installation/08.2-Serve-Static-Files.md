# 08.2 — Serve Static Files

> How to serve a static website (HTML/CSS/JS) via Nginx.
> Last updated: 2026-07-19

---

## The Problem

Nginx runs as user `www-data`. If your files are in `/home/flowero/`, Nginx **can't read them** because:

```bash
ls -la /home/
# drwxr-x--- flowero flowero /home/flowero/
#        ^^^
#  owner-only (750) — www-data is neither flowero nor in flowero group
```

---

## Step 1: Create the Nginx config

```bash
sudo nano /etc/nginx/sites-available/mysite.panomete.com
```

```nginx
server {
    server_name mysite.panomete.com;

    root /home/flowero/path/to/site;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }
}
```

---

## Step 2: Enable the site

```bash
sudo ln -s /etc/nginx/sites-available/mysite.panomete.com /etc/nginx/sites-enabled/
```

---

## Step 3: Fix permissions (the important part)

Nginx runs as `www-data` — it needs read access to your files:

```bash
# Add www-data to flowero's group
sudo usermod -aG flowero www-data

# Make files group-readable
chmod -R g+rX /home/flowero/path/to/site/
```

| Command | What it does |
|---------|-------------|
| `usermod -aG flowero www-data` | Adds `www-data` user to the `flowero` group. Now Nginx can traverse `/home/flowero/` |
| `chmod -R g+rX` | `g+r` = add read for group. `X` = add execute for directories only (needed to traverse). `-R` = recursive |

---

## Step 4: Test and restart

```bash
sudo nginx -t && sudo systemctl restart nginx
```

⚠️ **Must restart, not reload.** Group membership changes require a full restart — Nginx workers keep old group memberships until restarted.

---

## Step 5: Verify

```bash
curl -s -o /dev/null -w '%{http_code}' http://127.0.0.1 -H 'Host: mysite.panomete.com'
# Should return 200
```

---

## Quick Debug

| Symptom | Cause | Fix |
|---------|-------|-----|
| `403 Forbidden` | Nginx can't read the files | `sudo usermod -aG flowero www-data` + `chmod -R g+rX` |
| `404 Not Found` | File exists but Nginx can't access it | Same fix — restart nginx after group change |
| Works after `sudo` but not in browser | `www-data` has no permission | Same fix |

---

## Alternative: Copy to `/var/www/`

If you don't want to mess with home directory permissions:

```bash
sudo cp -r /home/flowero/path/to/site /var/www/mysite
sudo chown -R www-data:www-data /var/www/mysite
```

Then set `root /var/www/mysite;` in the Nginx config. No permission hacks needed.
