# 01.1 — Allow Other Devices to Remote

> How to authorize new devices to SSH into the server.
> Each device gets its own key pair — never share private keys.
> Last updated: 2026-07-19

---

## How It Works

```
Device (private key) → SSH connection → Server checks authorized_keys → Access granted
```

- **Private key** = stays on the device, never shared
- **Public key** = copied to the server's `authorized_keys`
- Each device has its own unique key pair
- Revoke access by removing the public key from the server

---

## Steps (on the new device)

### Step 1: Generate a key pair

```bash
mkdir -p ~/.ssh && chmod 700 ~/.ssh
ssh-keygen -t ed25519 -C "device-name" -f ~/.ssh/id_homelab -N ""
```

| Flag | What it does |
|------|-------------|
| `-t ed25519` | Modern key type — fast, short, secure |
| `-C "device-name"` | Label to identify this device (e.g., `friend-laptop`, `tablet`, `work-pc`) |
| `-f ~/.ssh/id_homelab` | Key file path |
| `-N ""` | No passphrase (for automation). Use a passphrase for personal devices |

### Step 2: Copy public key to server

**Option A: If you have the server password**

```bash
ssh-copy-id -i ~/.ssh/id_homelab flowero@remote.panomete.com
```

**Option B: Manual copy**

```bash
# On the new device — print the public key
cat ~/.ssh/id_homelab.pub

# On the server — add it
echo "ssh-ed25519 AAAA... device-name" >> ~/.ssh/authorized_keys
```

### Step 3: Verify

```bash
ssh -o BatchMode=yes flowero@remote.panomete.com "echo 'access granted ✅'"
```

---

## SSH Config (optional shortcut)

Create `~/.ssh/config` on the new device:

```
Host remote
    HostName remote.panomete.com
    User flowero
    IdentityFile ~/.ssh/id_homelab
```

Then just: `ssh remote`

---

## Revoke Access

Remove the device's public key from the server:

```bash
# On the server
nano ~/.ssh/authorized_keys
# Delete the line with the device's public key
```

That device can no longer SSH in.

---

## Quick Reference

| Action | Command |
|--------|---------|
| Generate key | `ssh-keygen -t ed25519 -C "name" -f ~/.ssh/id_homelab -N ""` |
| Copy to server | `ssh-copy-id -i ~/.ssh/id_homelab flowero@remote.panomete.com` |
| List keys on server | `cat ~/.ssh/authorized_keys` |
| Revoke access | Remove the key line from `authorized_keys` |
| Test connection | `ssh -o BatchMode=yes flowero@remote.panomete.com "echo ok"` |
