# 08.1 — Add a Subdomain

> How to add a new service to your Nginx reverse proxy.
> Last updated: 2026-07-19

---

## How It Works Now

```
Browser → Cloudflare → Tunnel → Nginx (:80) → routes by Host header → service
```

- **Cloudflare** handles TLS (HTTPS) at the edge — Nginx only receives HTTP
- **Tunnel** connects outbound from server to Cloudflare — no open ports needed
- **Nginx** routes traffic by the `Host` header (subdomain) to the correct service
- **DNS** is automatic — `*.panomete.com` already points to the tunnel

---

## Steps to Add a New Subdomain

### Step 1: Create the Nginx config

```bash
sudo nano /etc/nginx/sites-available/myservice.panomete.com
```

```nginx
server {
    server_name myservice.panomete.com;

    location / {
        proxy_pass http://127.0.0.1:PORT;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

Replace `PORT` with the port your service runs on (e.g., `3001` for Uptime Kuma, `8090` for Portainer).

### Step 2: Enable the site

```bash
sudo ln -s /etc/nginx/sites-available/myservice.panomete.com /etc/nginx/sites-enabled/
```

### Step 3: Test and reload

```bash
sudo nginx -t && sudo systemctl reload nginx
```

### Step 4: Verify

```bash
curl -s -o /dev/null -w '%{http_code}' http://localhost -H 'Host: myservice.panomete.com'
```

Should return the service's HTTP status code (200, 302, etc.).

---

## No DNS Changes Needed

`*.panomete.com` already routes to the tunnel. Any new subdomain works automatically — just add the Nginx config.

---

## Important: Bind Services to Localhost

Since Nginx handles external routing, bind your Docker services to `127.0.0.1`:

```yaml
# docker-compose.yml
ports:
  - "127.0.0.1:3001:3001"  # ✅ only accessible via Nginx
  # - "3001:3001"           # ❌ accessible from LAN/internet directly
```
