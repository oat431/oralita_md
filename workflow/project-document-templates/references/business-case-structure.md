# Business Case Template — Reference

## Canonical Structure (14 sections)

Based on BABOK v3 (Strategy Analysis) + PMBOK v8 (Initiating, ISO 21502).

| # | Section | Purpose | Key Tables |
|---|---------|---------|-----------|
| 1 | Executive Summary | One-page decision view | Summary table (Problem, Investment, Benefits, ROI, Risks) |
| 2 | Problem/Opportunity | What/Where/When/Who/Impact structured format | Problem statement table, Opportunity statement table |
| 3 | Business Objectives | SMART goals with strategic alignment | Objectives table (ID, Objective, Metric, Target, Deadline, Alignment) |
| 4 | Current State | People/Process/Technology/Data/Performance | Current state table, Pain points table, Current cost table |
| 5 | Options Analysis | ≥3 options with weighted scoring | Option summary table, Weighted scoring matrix (1-5 × weights) |
| 6 | Recommended Solution | Clear recommendation + scope boundaries | Solution overview table, In/Out scope table |
| 7 | Cost-Benefit | CAPEX/OPEX, NPV/IRR/ROI/payback | Cost tables, Benefits table, Financial metrics table, Cash flow projection, Sensitivity analysis |
| 8 | Risk Assessment | Business/Technical/Financial risks | Risk tables with Probability/Impact/Level/Mitigation, emoji heat map |
| 9 | Implementation | Phased timeline, resources, dependencies | Timeline table, Resource table, Dependencies table |
| 10 | Success Metrics | KPIs with baseline/target | KPI table, Benefits realization plan |
| 11 | Stakeholder Impact | Impact assessment + change readiness | Impact table, Readiness assessment |
| 12 | Assumptions & Constraints | Tracked assumptions, constraints, dependencies | Three separate tables |
| 13 | Recommendation | Decision requested + conditions + next steps | Decision summary table, Next steps table |
| 14 | Appendices | Financial model, research, glossary | Glossary table |

## Design Decisions

- **Weighted scoring matrix** (Section 5.3) uses percentage weights summing to 100%, scores 1-5 per criterion
- **Cash flow projection** (Section 7.4) is a year-by-year table with Investment/Operating Cost/Benefits/Net/Cumulative columns
- **Sensitivity analysis** (Section 7.5) tests 4 scenarios: benefits lower, costs higher, delayed start, discount rate change
- **Risk heat map** (Section 8.4) uses emoji table (Probability × Impact) with risk IDs placed in cells — NEVER ASCII art
- **Financial metrics** (Section 7.3) include NPV, IRR, ROI, Payback Period, Break-Even Point, Discount Rate, Analysis Period

## Source Standards

- BABOK v3 — Strategy Analysis (Current State, Future State, Change Strategy, Solution Scope, Gap Analysis, Risk Analysis, Potential Value)
- PMBOK v8 — Initiating (Business Case, Benefits Management Plan, Project Charter, Stakeholder Register)
- ISO 21502 — Project Management Guidance
- ISO/IEC/IEEE 12207 — Software Life Cycle Processes
